Euclidean Rhythms

Developed an application in vanilla JS to generate Euclidean Rhythms as part of a presentation for my college's STEM club. The application is drawn on a canvas and rendered through WebGL using Phaser. The sounds are played using Midi.js. These rhythms are generated using the bjorklund algorithm developed by George Toussaint. A detailed explanation can be read in the outline I wrote for my presentation here.

JavaRPG

A simple turn based rpg developed for my intro to programming class, made entirely with the Java standard library. As part of my requirements in developing the project I was required to display my understanding of OOP principles. Because of this abilities/skills and even AI are all extending from a base class that defines a simple interface for them to be given the data they need to operate on. Positions of characters are determined using an 8-bit bitmask. Key programming concerns when developing this application were extending the gradient class and drawing operations my professor had given me to work on any screen resolution.

Gentry

A strategy game developed in Javascript using the Phaser framework. Gameplay loops consisted of dialogue segment that were scripted using an extended version of the node editor developed by Evan Todd. His editor and implementation only supported parsing raw text due to him only needed to display simple text on the screen for his project. My extensions primarily consisted of adding new node definitions for audioclips/drawing sprites/utility and implementing a functionality for parsing mathematical expressesions and handling tasks such as substring coloring. The graph was represented using an adjacency list so I needed to roll my own parser using REGEX for finding closed brackets, a FSM(Finite State Machine) for handling node states and then using a recursive decent parser to handle the mathematical expressions. The rpg side of things was handled using a GOAP(Goal Oriented Action Planner) for enemies to choose their actions and the Command Queue design pattern for handling player actions. This means that each action chosen for a turn would be stored in Queue and would then be performed as elements are dequeued.
